### Security of Your Personal Data

The security of Your Personal Data is important to Us, but remember that no method of transmission over the Internet, or method of electronic storage is 100% secure. While We strive to use commercially reasonable means to protect Your Personal Data, We cannot guarantee its absolute security.

#### Technical Security Measures We Take
- SQLite databases are stored with restricted filesystem permissions
- Data is never transmitted to third-party servers (zero telemetry, zero analytics)
- No cloud storage, no external APIs called by the software
- Constant-time API key comparison (prevents timing side-channel attacks)
- Parameterized SQL queries (prevents SQL injection)
- Pairing codes are one-time use and expire after 5 minutes
- Bounded thread pool on the HTTP server (prevents DoS-based data exposure)
- SSRF protection (the Discord bot blocks connections to private/internal IP addresses)

#### Inherent Security Limitations You Acknowledge

1. **No Transport Layer Security**: The communication between the Discord bot and the Minecraft plugin uses **plain HTTP**, not HTTPS. API keys and whitelist data are transmitted in cleartext over the network. For production use, We recommend:
   - Running both services on the same machine or a local network
   - Using a VPN or tunnel (WireGuard, Tailscale, etc.)
   - Using a reverse proxy with TLS termination (nginx, Caddy)

2. **Server Operator Responsibility**: The security of Your data ultimately depends on the operator of the Discord bot and the Minecraft server. The Company has no control over:
   - Who has access to the server where the Bot runs
   - Whether the server operator follows security best practices
   - Whether the server operator shares or leaks data
   - Whether the server operator modifies the software

3. **Log Files**: The software may log operational information that includes usernames and IP addresses. Server operators should ensure log files are properly secured and rotated.
